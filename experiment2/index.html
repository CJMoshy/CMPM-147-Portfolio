Experiment 2 - Procedurally Generated Landscapes

Description

This experiment was done with an emphasis on the use of Perlin noise for procedural generation. What you are looking at is a sketch done with p5.js that depicts a desert landscape. This was done by using both Perlin noise and a custom noise function that uses a modified linear interpolation formula to sketch some jagged mountains.

Technical

The technicallity here is seen mainly in the two further back mountain ranges. These mountain ranges are generated using some of the basic principles of Perlin noise. Initially, a range of 2-dimensional vectors are generated, modeling an outline for the mountains. After this, the vectors are passed to a function which iterates them, using linear interpolation to calculate a mid point. This mid point is then adjusted with a 'noise' modifier, which adds some randomness to the mountains and gives them the signature jagged feel. Finally, p5.js is used to create a shape from the points. This shape is then filled with a color and rendered to the canvas. I approached this problem with an OOP style mindset, creating an object that handles all of the heavy lifting, with a render() function that can be called within draw() to render the mountains. The reason why I choose to use OOP here is because I think that it provided a clear way to organize the logic for creating a single mouintain range into one area of the codebase. In additon, an OOP approach allows me to set custom colors and parameters for each mountain range, and create multiple instances each with the aformentioned unique qualitites.

The closest mountain range is generated via 'true' Perlin Noise. This was done to model the differences between the algorithm I created and the Perlin algorithm. As you can see, the Perlin mouintains are much smoother. The main reason for this is becase my algorithim opts for linear interpolation to connect the intial points whereas Perlin noise models something much closer to cubic interpolation, which creaetes much smoother curves.

Reflection

Looking back on this assignment, I thoroughly enjoyed it. I appreciated the inspiration that was given to us by Professor Modes, with both the simplified and complex examples. In addition, I enjoyed doing a lot of the research about Perlin noise, as it's a technique that I have seen other developers use before and I was unsure about its inner workings. This assignment was also flexible enough to allow me to do custom work, furthering my knowledge in some basic procedural generation. If I were to change anything about the project, I would implement cubic interpolation alongside linear interpolation. This would provide much smoother curves for the mountains, and give more contrast to the types of mountains that could be generated. However, I am satisfied with how the linear interpolation turned out. It was a very interesting process learning about all of the different types of interpolation and the results and styles that they produce.
