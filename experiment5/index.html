Description

In this programming assignment, we're building a generator that directly reads inspiring images, renders parameterized images, and adjusts its own designs to optimize an evaluation score. My approach opts to implement custom algorithms that do all of the processing and generation.

Technical

The technicallity seen here is largely within the image processing algorithms. I opted to write custom processing algorithms that parse the p5 pixel-array into a more malleable format, returrning a 2d array of RGB values corresponding to each pixels color value. This was necessary because the main p5 pixel array is one dimensional and much more diffuclt to do any processing with.

The program works by first processing the pixel array and then parsing the processed array into a series of objects that represent a given color and x,y coordinate. This parsing is done by taking a square sub array from the processed pixels array and then averaging the RGB values within the square. The result is then returned to the rendering loop which will render a predetermined shape at the corresponding x,y location.

Reflection

I really wanted to challenge myself this time around. I could've very easily just used the templated example code and spent some time tuning the parameters to get something that worked and looked decent, but that didn't cut it for me. That's why I decided to write the entire application from scratch. Doing this provided me with some great algorithm practice and a more in-depth understanding of how someone might approach a problem like this.
