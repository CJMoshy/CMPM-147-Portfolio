Experiment 3 - Procedurally Generated Tilemaps

Description

This experiment is aimed at modeling the main algorithms behind procedural generation of tilemaps, specifically autotiling. All of the autotiling logic occours via a bitmask calculation made on specific tiles. If you want to test it for yourself, you can edit the top text box and add in 'w' tiles, or you can edit the bottom text box and add 'c' tiles. Enjoy!

Technical

As mentioned above, much of the technicallity here comes from the autotiling logic. The way this logic works is relatively straightforward. To begin, you pass in an index of an array which in this case correlates to the grid containing each tiles respective character refrence. After the index is read, it is passed into a function that will query around each of the cardinal directions for identical tiles, returning 1 or 0 respectivly. This result is then used to bitmask a final result value which is then passed into a lookup array. The lookup array contains offset values for the respective tile. Essentially what this means is that each tile has corresponding overlapping tiles that will be rendered on top of it in order to give it a less blocky feel.

Its also worth mentioning that the lake genration algorithm is a custom modification on the random walk algorithm. The properties of the random walk algorithm did suite the inital generation quite well, but left the lakes/rivers with holes inside and sometimes also produced very thin and awkward looking shapes. I slightly modified the algorithm to not only fill in these holes but to also have a tendency to ensure singular placements of 'w' characters would not occour.
